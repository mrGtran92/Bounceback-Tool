How to use: Upload both bounceback_checker.html and tests.html to the same GitHub repo. After any change to bounceback_checker.html, wait ~60 seconds for Pages to rebuild, then reload this page. All tests should show PASS. Any FAIL means a change broke a pathway — investigate before sharing the updated tool.

To add a scenario: Edit the SCENARIOS array in the script below. Each scenario needs: name, group, service, answers (the sequence of question/answer pairs that drive the tool), and expected (the result key string).

This page only works when served from GitHub Pages — it fetches bounceback_checker.html via a network request, which browsers block for local files. Do not open by double-clicking from your desktop.
